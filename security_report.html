Kritik strings.HasPrefix Mantık Hatası ile Dizin Geçişi (Path Traversal)
utils/pathutils.go – ResolvePath()
▼
Açıklama

ResolvePath fonksiyonu, çözümlenen dosya yolunun (absPath) izin verilen kök dizin (absRoot) sınırlarında kalıp kalmadığını strings.HasPrefix(absPath, absRoot) kontrolüyle denetler. Ancak kök dizinin sonuna bir dizin sınırlandırıcı (slash) eklenmediği için, aynı isim önekiyle başlayan yan dizinlere erişim sağlanabilmektedir. Örneğin root dizini /app/files ise, /app/files-secret/keys.txt dosyası bu kontrolü aşar çünkü /app/files-secret dizesi /app/files ile başlar.

Zayıf Kod Bloğu
Goutils/pathutils.go
func ResolvePath(rootDir, requestPath string) (string, error) {
    // ... yol temizleme işlemleri ...
    absPath = filepath.ToSlash(absPath)
    absRoot = filepath.ToSlash(absRoot)

    // ZAFİYETLİ KONTROL: absRoot sonuna "/" eklenmediği için yan klasörlere traversal yapılabilir
    if !strings.HasPrefix(absPath, absRoot) {
        return "", fmt.Errorf("path traversal detected: %s is outside root %s", requestPath, rootDir)
    }
    return filepath.FromSlash(absPath), nil
}
Çözüm Önerisi (Remediation)

absPath değerinin tam olarak absRoot değerine eşit olduğunu veya absRoot + "/" ile başladığını kontrol edin:

// GÜVENLİ KONTROL
if absPath != absRoot && !strings.HasPrefix(absPath, absRoot + "/") {
    return "", fmt.Errorf("path traversal detected")
}
Yüksek APIMediaStream Üzerinden Stored XSS ve SVG Enjeksiyonu
handlers/media.go – APIMediaStream()
▼
Açıklama

APIMediaStream endpoint'i medya dosyalarını oynatmak veya akış sağlamak amacıyla tasarlanmıştır. Ancak, istemcinin talep ettiği dosyanın uzantısına yönelik hiçbir sınırlandırma (whitelist) uygulanmamaktadır. Bir saldırgan HTML dosyası veya içinde zararlı JavaScript kodu barındıran SVG dosyası yükleyebilir. Bu SVG veya HTML dosyası akış API'si üzerinden istendiğinde, sunucu Content-Type: image/svg+xml veya text/html header'ı ile yanıt dönecek ve tarayıcı JavaScript kodunu kurbanın oturum bağlamında çalıştıracaktır.

Zayıf Kod Bloğu
Gohandlers/media.go
func APIMediaStream(c *fiber.Ctx) error {
    // ... yetki ve path çözümleme ...
    fileSize := stat.Size()
    mimeType := utils.GetMimeType(filePath) // Güvenli olmayan MIME ataması (örn: text/html)

}
Çözüm Önerisi (Remediation)

APIMediaStream içindeki dosya türlerini sadece geçerli ses ve video formatları ile sınırlandırın (Örn: mp3, mp4, webm, wav). SVG ve HTML dosyalarının bu kanal üzerinden sunulmasını kesinlikle engelleyin. Ayrıca genel dosyaları sunarken Content-Disposition: attachment header'ını kullanarak dosyaların tarayıcıda çalışmasını engelleyin.

Orta Dosya Yükleme İşleminde Uzantı & Tip Filtrelemesi Eksikliği
handlers/file.go – APIUploadFile()
▼
Açıklama

Uygulamanın dosya yükleme API'si (APIUploadFile), yüklenen dosyaların uzantılarını veya içerik tiplerini (MIME Type) herhangi bir filtrelemeye (whitelist/blacklist) tabi tutmamaktadır. Her ne kadar Go tabanlı backend bu dosyaları sunucu tarafında doğrudan çalıştırmasa da, bu durum Stored XSS saldırılarına, sunucu disk alanının doldurulmasına ve potansiyel olarak sunucu önüne konulabilecek bir Nginx/Apache konfigürasyon hatasında Remote Code Execution (RCE) zafiyetlerine zemin hazırlar.

Zayıf Kod Bloğu
Gohandlers/file.go
func APIUploadFile(c *fiber.Ctx) error {
    // ... yetkilendirme ve form alma ...
    files := form.File["files"]
    for _, file := range files {
        destPath := filepath.Join(uploadPath, file.Filename)
        fullDest, err := utils.ResolvePath(rootDir, destPath)

        }
    }
}
Çözüm Önerisi (Remediation)

Uygulamanın amacına göre bir dosya uzantısı izin verilenler listesi (Whitelist) oluşturun. Tehlikeli uzantıların (.html, .htm, .svg, .jsp, .asp, .php, .exe, .sh, .bat) yüklenmesini engelleyin veya yüklenen dosyaları kullanıcıya sunarken kesinlikle tarayıcıda çalışmayacak şekilde (attachment olarak) yapılandırın.

Orta CSRF Token'ının Oturum ile İlişkilendirilmemesi (Double Submit)
auth/middleware.go – CSRFMiddleware()
▼
Açıklama

Uygulama, CSRF saldırılarını engellemek için Double Submit Cookie desenini kullanmaktadır. Ancak, csrf_token çerezi veritabanındaki kullanıcı oturumuyla (session) hiçbir şekilde ilişkilendirilmemiştir. Sadece istemci çerezindeki değer ile istek başlığındaki (X-CSRF-Token) değerin eşleşip eşleşmediği kontrol edilmektedir. Eğer saldırgan, hedef alan adının herhangi bir alt alan adını (subdomain) kontrol edebiliyorsa veya Man-in-the-Middle (MITM) / XSS yoluyla kurbana yapay bir CSRF çerezi enjekte edebiliyorsa, bu korumayı kolayca atlatabilir.

Zayıf Kod Bloğu
Goauth/middleware.go
func CSRFMiddleware(c *fiber.Ctx) error {
    csrfToken := c.Cookies("csrf_token")
    // ... token oluşturma ve set etme ...
    
    // ZAFİYETLİ KONTROL: Token session bazlı doğrulanmıyor, sadece cookie ve header eşitliği aranıyor
    requestToken := c.Get("X-CSRF-Token")
    if requestToken == "" || requestToken != csrfToken {
        return utils.SendError(c, fiber.StatusForbidden, "Geçersiz CSRF token")
    }
    return c.Next()
}
Çözüm Önerisi (Remediation)

CSRF token'larını veritabanındaki veya hafızadaki session nesneleri içine kaydedin. İstek geldiğinde, form veya header ile gönderilen CSRF token değerini cookie ile değil, kullanıcının aktif oturumundaki (Session tabanlı) token değeri ile karşılaştırın.

Orta Dosya Arama API'sinde Disk I/O Tüketimi (Resource Exhaustion)
handlers/search.go – APISearch()
▼
Açıklama

APISearch fonksiyonu arama işlemi yaparken disk üzerindeki tüm dizin ağacını filepath.Walk fonksiyonu ile baştan sona taramaktadır. Arama işlemi veritabanı indeksleri yerine doğrudan disk I/O operasyonuyla gerçekleştirilmektedir. Geniş dosya sistemlerinde veya çok sayıda dosya barındıran sunucularda, bu endpoint'e gönderilecek ardışık veya eş zamanlı arama istekleri CPU ve disk I/O limitlerini tüketerek sunucunun kilitlenmesine (Denial of Service - DoS) neden olur.

Çözüm Önerisi (Remediation)

Dosya sistemini periyodik olarak arka planda indeksleyen bir yapı kurun ve aramaları bu indeks (bellek veya SQLite) üzerinden gerçekleştirin. Alternatif olarak, arama derinliğini sınırlayın ve arama işlemleri için çok daha katı bir rate-limiting uygulayın.

Düşük Oturum Çerezlerinde 'Secure' Bayrağının Eksikliği
auth/auth.go – HandleLogin() / HandleLoginAPI()
▼
Açıklama

Oturum doğrulaması için kullanılan session_token çerezi oluşturulurken Secure: false olarak set edilmiştir. Bu durum, uygulama HTTPS (SSL/TLS) protokolü üzerinden çalıştırılsa bile, çerezin tarayıcı tarafından şifrelenmemiş HTTP bağlantıları üzerinden de gönderilmesine neden olur. Bu durum ağ trafiğini izleyen bir saldırganın (Man-in-the-Middle) oturum çerezini ele geçirmesini kolaylaştırır.

Zayıf Kod Bloğu
Goauth/auth.go
c.Cookie(&fiber.Cookie{
    Name:     "session_token",
    Value:    session.Token,
    Path:     "/",
    MaxAge:   7 * 24 * 60 * 60,
    HTTPOnly: true,
    Secure:   false, // UYARI: HTTPS altında bile çerez güvensiz iletilebilir
    SameSite: "Lax",
})
Çözüm Önerisi (Remediation)

Çerez ayarlarında Secure parametresini true yapın. Eğer uygulamanın lokalde (HTTP) geliştirilme aşamasında çalışması gerekiyorsa, bunu konfigürasyon dosyasından (örneğin bir config parametresi veya Environment Variable ile) dinamik olarak yönetin.

Düşük Windows İşletim Sistemlerinde Backslash (\) Kontrol Eksikliği
services/trashservice.go – RestoreFromTrash()
▼
Açıklama

Çöp kutusundan dosya silme veya geri yükleme işlemleri esnasında kullanılan trashID parametresi, dizin geçişlerini engellemek için strings.Contains(trashID, "/") ve ".." kontrollerinden geçirilir. Ancak Windows işletim sistemlerinde ters slash (\) karakteri de geçerli bir dizin ayıracıdır ve bu karakter filtrelerde yer almamaktadır. Her ne kadar .. kontrolü sayesinde üst dizine çıkılması zorlaştırılsa da, Windows sunucularda beklenmeyen alt dizin yönlendirmelerine sebebiyet verebilir.

Zayıf Kod Bloğu
Goservices/trashservice.go
func RestoreFromTrash(trashBaseDir, rootDir, username, trashID string) error {
    // ...
    // EKSİK FİLTRE: Windows sistemlerdeki "\\" kontrolü yapılmamıştır
    if strings.Contains(trashID, "..") || strings.Contains(trashID, "/") {
        return fmt.Errorf("invalid trash ID")
    }
    // ...
}
Çözüm Önerisi (Remediation)

Zafiyetli satırı Windows dizin ayracını da kapsayacak şekilde güncelleyin:

if strings.Contains(trashID, "..") || strings.Contains(trashID, "/") || strings.Contains(trashID, "\\") {
    return fmt.Errorf("invalid trash ID")
}
